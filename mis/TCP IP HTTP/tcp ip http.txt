TCP/IP的分层管理
	 应用层：负责处理特定的应用程序，如FTP、DNS、HTTP等。DNS(Domain Name System),域名解析系统
	 传输层：对上层应用层提供处于网络连接中两台计算机之间的数据连接。该层有两种协议：TCP和UDP。
	 网络层：处理网络上流动的数据报。该层协议有：IP、ICMP、IGMP。
	 数据链路层：处理连接网络的硬件部分。包括操作系统、硬件设备驱动、网卡、光纤等

TCP三次握手原理：SYN：同步序号 ACK：应答回复 FIN：发送方完成数据发送
	
		  clinet 		server                                          状态
第一次    syn=j 
第二次                 向client确认请求ack=j+1  请求syn=k               SYN_RECV
第三次    ack=k+1                                                       ESTABLISHED

三次握手的目的：消除旧有连接请求的SYN消息对新连接的干扰，同步连接双方的序列号和确认号并交换TCP 窗口大小信息
-----------------------------------------------------------------------------------------------------------------------------------
需四次挥手原因：由于TCP的半关闭特性，TCP连接时双全工（即数据在两个方向上能同时传递），因此，每个方向必须单独的进行关闭

		  A 		         B                                          状态
第一次    fin=j                                                         请求断开A->B
第二次                 向client确认请求ack=j+1  请求fin=k               断开A->B
第三次                 fin=k                                            请求断开B->A
第四次    ack=k+1                                                       断开B->A
